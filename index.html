HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The head element is not seen by the user apart from the Title which appears in the browser tab.
  • The metadata in the head element tells the browser what charset is used, as well as inital viewport settings to ensure the website displays equally as well on desktop and mobile. It also defines http-equiv which I'm not currently sure of the purpose
  • The body element is made up of a header element, a main element and a footer element.
  • The header element typically contains page title and navigation menus.
  • The main element contains the bulk of the content of the page.
  • The footer element contains details to go at the bottom of the page, in this case a simple footnote but could also contain copyright details, or contact details I suppose

Git

  • git status: checks what branch we are currently on and whether there are uncommitted changes as well as any updates to pull from the remote repository.
  • git checkout (branchname): switches to an existing branch to be worked on.
  • git checkout -b (branchname): creates a new branch and switches to it.
  • git add -A: stages all changes ready to be committed.
  • git commit -m "commit message": commits staged changes to the current branch.
  • git pull origin main: pulls any updates from the main branch into the current branch.
  • git push origin (branchname): pushes changes into the current branch.
  • Feature branches should be named feature/(newfeaturename) to mark them clearly as feature branches.
  • When a feature branch is complete it can be merged into the main branch using a Pull Request.
  • Pull Requests can have reviewers assigned to them to have a second set of eyes check the code prior to merging into main.
